## 1. Reservoir-storage — bulletproofing + quality

**The reconciliation is stubbed.** `docs/survey-notes.md` claims the reconcile is "done (2026-06-21)", but the persisted `data/audit/reconcile.json` was generated from placeholder values — `GRERESCO` expected `1.0` with `got_af` null, `blue-mesa` expected vs got off by ~50% (`match=false`), and a literal `NOTREAL` test reservoir. The notebook's `expected` block is commented out, so the live check is disabled by default. `AGENTS.md` is the only honest line ("`reconcile()` has no expected totals filled yet"). **There is zero passing reconciliation on record** — and per the bulletproofing checklist that reconciliation *is* this project's canonical pre-publication gate. → [#38](https://github.com/CUPIDS-Lab/co-environmental-data/issues/38)

**Bulletproofing checklist:**

| Item | Status | Evidence |
| --- | --- | --- |
| Record counts vs expected | ⛔ FAIL | CSV has **53 reservoirs**, not the 138 enumerated; the ~85 missing returned 404/no-data but no doc reconciles the shortfall, so a silent drop is indistinguishable from real no-data. → [#41](https://github.com/CUPIDS-Lab/co-environmental-data/issues/41) |
| Ranges/dates plausible | ⛔ FAIL | Impossible values shipped: elevation **70,235 ft** (Dillon), **2,903 negative elevations**, **134 negative `release_cfs`**. Audit value-range table shows them; nothing gates them. → [#40](https://github.com/CUPIDS-Lab/co-environmental-data/issues/40) |
| Missing vs zero vs sentinel | 🟡 PARTIAL/FAIL | `reclamation_rise.py` `attrs.get("result") or attrs.get("value")` turns a real **0.0 storage into NA** (~5,217 zeros / ~7,451 NAs, boundary corrupted). → [#39](https://github.com/CUPIDS-Lab/co-environmental-data/issues/39) |
| GROUP BY for dupes/spelling | 🟡 PARTIAL | Composite-key uniqueness enforced, but `Blue Mesa Reservoir` (RISE) vs `BLUE MESA RESERVOIR` (DWR) — no crosswalk. → [#41](https://github.com/CUPIDS-Lab/co-environmental-data/issues/41) |
| Recompute a 2nd way / agree | ⛔ FAIL | The reconcile (the canonical recompute) is stubbed/failing. → [#38](https://github.com/CUPIDS-Lab/co-environmental-data/issues/38) |
| Hand-check random sample | ⛔ undocumented | No record of a passing by-hand spot-check. |
| Independent replication | ◻ not started | Depends on onboarding a second reviewer ([#31](https://github.com/CUPIDS-Lab/co-environmental-data/issues/31)/[#32](https://github.com/CUPIDS-Lab/co-environmental-data/issues/32); CODEOWNERS still placeholders). |
| Data notebook of procedures | 🟡 PARTIAL | Exists (decision-log + survey-notes + notebook) but **inaccurate** (records reconcile as done when it isn't), so it fails the "defend" purpose. → [#38](https://github.com/CUPIDS-Lab/co-environmental-data/issues/38) |

**Data-quality (new defects, all → [#41](https://github.com/CUPIDS-Lab/co-environmental-data/issues/41) unless noted):** `qa_flag` join emits the literal string `"None None"` in ~268k rows (`"O None"` in ~134k); no **Known issues** section in `docs/data-dictionary.md`; false precision (elev to 0.01 ft) unassessed.

**Strong / DONE (no action):** vertical-datum + capacity-baseline caveats documented (`concepts.yaml`); multi-reading-per-day flooring documented + tested; immutable originals + sha256 provenance; tidy-long pandera contract; idempotent pipeline + monthly CI; 30 passing offline tests.

**Doc inaccuracies corrected in this PR:** `README.md`/`AGENTS.md` "17 tests" → **30**; `survey-notes.md` false "reconcile done" claim removed; a Known-issues section added to the pipeline data dictionary.
