## Bulletproofing (probe-the-source; reconcile vs truth)

- **streamflow** — ✅ `audit.reconcile_cross_source` joins USGS↔DWR (DWR re-serves many USGS gages) on `(usgs_site_no, date)`; the two independent series agree 97.6–100% on shared dates (median 100%) — a genuine independent accuracy check. `audit.reconcile` additionally spot-checks the latest value vs the agency page (operator-supplied `expected`). Negative/ice sentinels → NA.
- **snowpack** — ✅ `audit.basin_percent_normal` computes basin SWE as % of the 1991–2020 day-of-year normal (the figure NRCS publishes), with `reconcile_basin_pct_normal` as the accuracy gate vs the NRCS basin report; `colocated_pairs` + `reconcile_cross_source` check co-located SNOTEL↔snow-course agreement (a looser, nearby-not-identical check, 86–100% in the live Gunnison sample). Negatives → NA; missing ≠ zero.
- **climate-stations** — ⚠️ **single authoritative source** (CO DWR/CDSS), so there is no independent series to reconcile against; QA is internal (schema + range/continuity + non-negative sentinel cleaning, temperatures excluded). *Recommended follow-up:* an optional cross-check of CDSS `SnowSWE`/precip vs NOAA GHCN for co-located stations (a real independent corroborator) — tracked as a future enhancement, not a blocker.

## Data quality (schema, ranges, missingness, tests)

- **All three** validate at the boundary with a **pandera** contract (`schema.validate`) enforcing the controlled `variable` vocabulary, unit set, and composite-key uniqueness `(source, site_id, datetime, variable)`; `co_pipeline_core.schema.normalize_long` day-floors timestamps and collapses duplicate keys to the latest reading. NA is preserved (never zero-filled); impossible negatives are mapped to NA per the non-negative-domain rule.
- **Test coverage (offline, fixture-based):** streamflow **21**, snowpack **28**, climate-stations **23** — parsers tested against trimmed real API responses, plus schema-contract, fetch no-data, and multi-source integration tests. A CI gate now asserts each README's stated test count matches reality.
- **Value ranges** are surfaced per variable by `audit.audit_processed` on every run (min/median/max) for outlier review; no impossible values were observed in the live samples.
